報告書 AI 預審系統 — 完整審閱流程

三層知識庫審閱架構｜公司審核規範 → Karen 逐段原則 → Ed/Jerry 方向性預警

1

① PM 初版撰寫

PM 依專案需求撰寫報告書初稿（服務建議書或執行報告書）。
👤 PM 📄 初版 .docx
2

② KK（直屬主管）審閱建議

Karen 使用 Word 追蹤修訂 + 插入評論，逐字逐句細節修改。關注文字精準度、段落邏輯、圖表規範、數據一致性。
📄 初版 .docx 📝 追蹤修訂 .docx 👤 Karen
3

③ 彙整為內審版

PM 接受追蹤修訂、回應評論、完成修改，彙整為「內審版」，準備送大主管審閱。
📝 KK 修訂 📄 內審版 .docx 📑 內審版 .pdf 👤 PM
AI

🤖 AI 預審系統介入（本系統）

在送大主管之前，自動執行三層預審。讓大主管能聚焦在高層次判斷，不需反覆指出同樣的錯誤。
📄 .docx + 📑 .pdf 🧠 三層知識庫 📊 審閱報告 ✅ 待辦清單

▼ 三層審閱執行順序

L0 公司審核規範 checklist（基線）
↓
L1 Karen 撰寫原則（逐段比對 + 名詞規範）
↓
L2 Ed / Jerry 方向性預警（大方向 + 委員質疑）
↓
PM PM 自檢提醒（AI 無法判斷的主觀項目）

▼ 雙軌分析：docx + PDF 互補

📄 docx XML 分析
字型設定、交互參照、隱藏式超連結、段落樣式、孤行寡行控制
📑 PDF 渲染分析
圖表標號跨頁、版面留白、雙面列印奇偶頁、實際頁碼定位
4

④ Ed、Jerry（大主管）審閱建議

大主管以純文字清單提供方向性建議（標示頁碼）。AI 預審已先排除基礎問題，大主管可聚焦高層次判斷。
📊 AI 審閱報告（參考） 📑 內審版 .pdf 📝 Ed 建議 .txt 📝 Jerry 建議 .txt 👤 Ed / Jerry
5

⑤ 完整版（最終定稿）→ 對外送件

PM 彙整所有修改意見，產出最終定稿（目標 85-95 分水準），送件出去。
📝 大主管建議 ✅ AI 待辦清單 📄 完整版 .docx / .pdf 👤 PM
🔄

🔄 AI 校準點（持續改進迴路）

用完整版反向校準評分規則、驗證 Ed/Jerry 建議落實率、提煉正面範例到知識庫。
目前已完成 8 份報告校準，全部落在 85-95 分預期區間。
📄 完整版 ⚙️ 評分規則調整 📚 知識庫更新 ⭐ 正面範例庫

L0 公司審核規範 checklist（基線）

每次審閱必先執行，依文件類型載入對應區塊
📖 知識庫 公司審核規範_通則.md
🔀 文件類型分流 A 全適用 + B 服務建議書 / C 期中期末 / D 期初
  • 📄 A. 基本樣式審查：封面、目錄、圖表參照、字型、樣式一致性 docx XML
  • 📑 圖表標號不跨頁：圖/表標題與本體須在同一頁 PDF bbox
  • 📄 交互參照完整性：所有圖表編號可正確連結 docx XML
  • 📄 隱藏式超連結：超連結必須顯示完整 URL docx XML
  • 📑 版面留白 / 雙面列印：無過多空白頁、奇偶頁正確 PDF
⬇

L1 Karen 撰寫原則（逐段比對）

基線通過後，逐章節比對直屬主管的撰寫原則與名詞規範
📖 主知識庫 撰寫原則_Karen.md
📖 名詞規範 名詞規範總表.md + 客戶層名詞
📎 條件載入 服務建議書 → Section F
⭐ 參照 正面範例庫.md
  • 🔤 名詞比對：記錄/紀錄、計劃/計畫、優化、模版/模板、污染/汙染…等 全文搜索
  • ✏️ 文字精準度：錯字（介皆→介接）、重複字（將將）、口語化（壓測→壓力測試）docx 文字
  • 📐 段落邏輯：過度使用「並」、人稱代詞「我們」、用字一致性 docx 文字
  • 📊 圖表規範：標號連續性、標題格式、表格完整性 docx + PDF
⬇

L2 Ed / Jerry 方向性預警

大主管視角的高層次問題，以「預警」形式提醒而非逐字修改
📖 Ed 知識庫 撰寫原則_Ed.md
📖 Jerry 知識庫 撰寫原則_Jerry.md
❓ 委員質疑 委員質疑清單_Jerry.md
📋 客戶備註 客戶_XXX/審閱備註_*.md
Ed 視角
  • 👁️ 閱讀體驗與整體流暢度
  • 📊 量化數據是否充分
  • 🎨 視覺呈現與版面
Jerry 視角
  • ⚙️ 技術正確性與術語
  • 🏗️ 章節結構完整度
  • ❓ 委員必問議題預防
⬇

PM PM 自檢提醒

AI 確實無法判斷的項目，歸入 PM 人工確認，並說明原因
  • 🖼️ 圖片解析度：主觀畫質判斷，AI 無法評估
  • 📊 圖表數據一致性：需人工交叉比對內文數字與圖表
  • 👤 截圖中人名：圖片內嵌文字無法程式讀取
  • 🔗 外部連結有效性：需人工點擊確認可開啟
⬇
